​首版核心功能需求文档（MVP 1.0）​
目标：基于JQData数据源，快速上线一个本地化部署的中国股市量化平台，聚焦核心功能闭环（数据展示、策略回测、模拟交易），确保后续扩展性。

​一、核心功能模块（MVP 1.0）​
​1. 数据展示与分析
需求优先级：最高

​基础功能：
​实时行情看板：展示沪深A股、ETF的实时价格、涨跌幅、成交量，支持按行业/市值/涨跌幅排序。
​财务数据展示：个股的PE、PB、ROE等核心财务指标，数据来源于JQData。
​技术实现：
前端使用ECharts渲染图表，后端通过JQData API拉取数据并缓存至本地MySQL。

​2. 策略回测引擎
需求优先级：高

​基础功能：
​Python策略开发：支持用户上传Python策略脚本，调用JQData历史数据（日线/分钟线）进行回测。
​绩效指标计算：输出年化收益率、夏普比率、最大回撤、胜率等核心指标。
​参数优化：支持网格搜索优化策略参数（如均线周期）。
​技术实现：
回测引擎采用事件驱动架构，基于Backtrader框架二次开发，集成JQData数据接口。
​3. 交易模拟系统
需求优先级：中高

​基础功能：
​虚拟账户管理：初始资金配置（默认100万），支持多策略独立运行。
​订单执行模拟：限价单/市价单撮合，模拟涨跌停限制、滑点（固定0.1%）。
​交易记录与持仓：展示成交明细、持仓盈亏、资金曲线。
​技术实现：
订单撮合逻辑基于JQData实时行情模拟，风控规则（如涨跌停）通过本地规则引擎实现。
​4. 用户权限与数据合规
需求优先级：中

​基础功能：
​本地账户体系：角色分级（管理员/普通用户），操作日志记录。
​数据授权验证：确保JQData接口调用合规，禁止数据二次分发。
​二、技术架构设计（MVP 1.0）​
​分层架构：

​前端：Vue.js + ECharts（数据可视化）。
​后端：Django REST Framework（API接口） + Celery（异步任务调度）。
​数据层：MySQL（结构化数据） + Redis（缓存行情数据）。
​量化引擎：Python（策略逻辑） + C++（高性能回测核心）。
​扩展性预留：

​模块化设计：数据接入、回测引擎、交易模拟模块独立，后续可替换数据源（如聚宽）或升级撮合引擎。
​API预留：开放策略部署、数据导出接口，为后续实盘交易对接预留能力
